Why Functions Exist

Understand the problem before learning the syntax.

❌ Repetitive Code
print("Hello, Alice") print("Hello, Bob") print("Hello, Carol") print("Hello, Dana")

The greeting logic is repeated four times. Changing "Hello" to "Hi" requires editing every single line.

✅ Organized Code
def greet(name): print("Hello, " + name) greet("Alice") greet("Bob") greet("Carol")

The greeting logic lives in one place. Change it once and every call updates automatically.

Key insight: A function is a named block of instructions you write once and run as many times as needed.

Define vs Call

Defining stores instructions. Calling runs them. Completely different actions.

DEFINING
def greet(): print("Hello")

Python reads and stores the instructions under the name greet. Nothing runs. No output.

CALLING
greet()

Tells Python: "find greet and run it now." Output happens here.

print vs return — And How result Works

Where most beginners get confused. We will make it completely clear.

print() — Displays
def show(a, b): print(a + b) show(3, 5) # displays 8 r = show(3, 5) print(r) # prints: None
  • Shows the value on screen
  • Value cannot be stored or reused
  • Assigning it gives you None
return — Sends Back
def add(a, b): return a + b result = add(3, 5) print(result) # displays 8
  • Sends the value back to calling code
  • Value can be stored in a variable
  • result holds 8 — reusable

Parameters & Inputs

Functions can receive information. Parameters are how that works.

Parameter = placeholder variable in the definition.  |  Argument = actual value passed in the call.
No parameters
def greet(): print("Hello") greet()

Empty () — no input needed. Always does the same thing.

One parameter
def greet(name): print("Hi, " + name) greet("Alice")

name = parameter. "Alice" = argument.

Two parameters
def add(a, b): return a + b add(3, 5)

Matched by position: a=3, b=5

Fix the Confusion Zone

The exact misunderstandings most beginners have. Click each to reveal the correction.

❓ "If I define a function, doesn't it already run?" ▶
No. Defining only stores the instructions. Python reads them but does not execute them. No output, no action — until you explicitly call the function by name with parentheses.
❓ "If I return 8, why don't I see 8 on screen?" ▶
return sends the value back to the calling code — it does not display anything. To see 8, you must explicitly print it: print(add(3,5)) or store it then print: result = add(3,5) then print(result).
❓ "Is result part of Python?" ▶
No. result is a variable name you chose. You could name it anything: answer, total, x. It is a storage container you create with the = operator. Python does not know about result until you write it.
❓ "Can I use any name after def?" ▶
Almost. The name must follow Python identifier rules: start with a letter or underscore, no spaces, no special characters, and cannot be a Python keyword (like def, return, print, if).
❓ "Is add already built into Python?" ▶
No. add is a function you defined. Python has built-ins like print, len, and type — but add is yours. Before def add(a, b):, Python has no idea what add is.
❓ "What happens if I forget () when calling a function?" ▶
Without parentheses, you are not calling the function — you are referencing it as a value. Writing greet instead of greet() does not run it. No code executes.
❓ "What if I define a function and never call it?" ▶
Nothing runs. The function exists in memory but its instructions never execute. No output, no computation, no error. Every function must be explicitly called for it to do anything.
❓ "What is the difference between add and print?" ▶
print is a built-in Python function that already exists when Python starts. add is a function you defined yourself. Both are called the same way — with () — but print is built-in and add is user-defined.
